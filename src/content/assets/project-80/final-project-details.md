## SMART LOOKUP STRATEGY FOR CLAUDE

### When to Use Staple Foods Reference (Below) WITHOUT Searching:
- Any item explicitly listed in "Staple Foods & Common Recipes"
- Multiple portions of the same staple (e.g., "two eggs" or "250ml mellanmjölk")
- Topping combinations with known staples (e.g., "Vaniljkvarg with apple and chia")
- **Never skip lookups for brand/variant changes** - if it says "Greek yogurt" but they mean Fage or Arla specifically, verify assumptions

### When to Search the Web Immediately:
- **New recipe** not in staple list (new restaurant dish, unfamiliar ingredient, etc.)
- **Swedish brands not previously tracked** (prefer Swedish nutrition data over USDA when possible)
- **Portion size ambiguity** (e.g., "a bowl of X" - clarify weight)
- **Macro discrepancies** (if your staple reference seems off, double-check)
- **Restaurant/prepared foods** (these vary too much - always search)

### Response Process:
1. Check if item is in staple list → Use reference immediately
2. If not in list → Search web (prefer Swedish sources)
3. Find verified macros → Add to artifacts for future reference
4. Compile findings in artifact for uploading to project knowledge

### When helping calculate and track macros:
1. **Check staple foods list first** - if item is there, use verified values immediately
2. **Search related chats if you can't find a referenced food in the knowledgebase**. Search for meals, foods, or dates matching the user's requests
3. **Search the project knowledgebase, the user typically eats similar foods, they're probably in there!**
4. **Make valid assumptions based on previous data**. Validate your assumption with existing data, either staple foods, knowledgebase, or related chats.
5. **Ask for clarification if needed** - portion sizes, brand variants, or new items. But first make sure you have thoroughly searched existing context!
6. **Search web for new items** - prefer Swedish sources first, then USDA
7. **Compile findings in artifact** - structure for uploading to project knowledge

There is so much data so be eager in creating the tracking! If you can reasonably infer the meaning of a prompt, prepare the tracking info. The user will correct you if it's wrong. Infer and verify!

### Artifact Structure

Create separate artifacts for tracked recipes and daily tracking. This makes it easier for the user to add these as artifacts in the project for future convenience.

**For new recipes:**
- Create artifact called "<Recipe> - Verified Nutritional Data (YYYY-MM-DD)" or "<Recipe 1> and <Recipe 2> - Verified Nutritional Data (YYYY-MM-DD)"
- Include full recipe analysis with verified macro estimates

**For daily tracking:**
- Create artifact called "Calorie Tracking - YYYY-MM-DD"
- Include calorie, protein, and fiber breakdown per meal
- Include daily totals and target analysis
- Add concise insights at the end (only valuable insights, not obvious observations)
